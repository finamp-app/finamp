# Android Auto — Recently Played Albums

## Overview

Add a new **Recently Played Albums** root node to the Android Auto browse tree. It shows a flat list of up to 20 albums sorted by most recently played. Tapping an album plays it immediately without drilling into a sub-list. Not available in offline mode.

---

## Scope

- One new root browse node (Albums only)
- Online only — offline shows a "Not available offline" placeholder
- No pagination (20-item cap fits comfortably within the Binder IPC limit)
- No letter browsing (irrelevant for a short recently-played list)
- Localisation key added for the node title (English fallback string used until translations land)

---

## Architecture

### 1. Model — `MediaItemParentType` enum

Add `recentlyPlayed` as `@HiveField(3)` to the existing enum in `lib/models/finamp_models.dart`:

```dart
enum MediaItemParentType {
  @HiveField(0) collection,
  @HiveField(1) rootCollection,
  @HiveField(2) instantMix,
  @HiveField(3) recentlyPlayed,
}
```

No new fields on `MediaItemId`. Regenerate `lib/models/finamp_models.g.dart` via `build_runner` after the change.

### 2. Root menu — `music_player_background_task.dart`

Add one node to `_getRootMenu()`, after the existing Artists node:

```dart
MediaItem(
  id: MediaItemId(
    contentType: TabContentType.albums,
    parentType: MediaItemParentType.recentlyPlayed,
  ).toString(),
  title: _appLocalizations?.recentlyPlayedAlbums ?? "Recently Played Albums",
  playable: false,
),
```

No `CONTENT_STYLE_*` extras — list view (default) is appropriate for a short recently-played list.

### 3. Data fetching — `android_auto_helper.dart`

Add a branch in `getMediaItems()`:

```dart
if (itemId.parentType == MediaItemParentType.recentlyPlayed) {
  return _getRecentlyPlayedItems(itemId);
}
```

New private method `_getRecentlyPlayedItems(MediaItemId itemId)`:

**Offline:** returns a single non-playable placeholder.

**Online:** Jellyfin only tracks `DatePlayed` on individual tracks, not on album items. Two-step fetch:

1. Fetch up to 100 recently played tracks (`SortBy=DatePlayed,Descending,Filters=IsPlayed`) — `Filters=IsPlayed` works correctly for tracks.
2. Deduplicate by `albumId`, preserving order, take first 20 unique IDs.
3. Fetch those 20 albums by ID, re-sort to match play order, map via `generateMediaItem` with `parentType: collection`.

### 4. Playback — no changes needed

Tapping a recently-played item calls the existing `playFromMediaId` path. Items with `parentType: collection` and a valid `itemId` already route through `getBaseItems(itemId)` → `queueService.startPlayback(...)`. Albums and artists both pass `_isPlayable`, so no guard changes are required.

## Out of scope

- Offline fallback via local `PlaybackHistoryService` (session-only, track-level — not worth the complexity for a first pass)
- Pagination (20 items is sufficient and avoids IPC complexity)
- Letter browsing on the recently-played list
- Recently played artists, tracks, or playlists
