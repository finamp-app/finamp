# Android Auto — Recently Played Albums & Artists

## Overview

Add two new root nodes to the Android Auto browse tree: **Recently Played Albums** and **Recently Played Artists**. Each shows a flat list of up to 20 items sorted by most recently played (server-side Jellyfin data). Tapping an item plays it immediately without drilling into a sub-list. Not available in offline mode.

---

## Scope

- Two new root browse nodes (Albums, Artists)
- Online only — offline shows a "Not available offline" placeholder
- No pagination (20-item cap fits comfortably within the Binder IPC limit)
- No letter browsing (irrelevant for a short recently-played list)
- Localisation keys added for both node titles (English fallback strings used until translations land)

---

## Architecture

### 1. Model — `MediaItemParentType` enum

Add `recentlyPlayed` as `@HiveField(3)` to the existing enum in `lib/models/finamp_models.dart`:

```dart
enum MediaItemParentType {
  @HiveField(0) collection,
  @HiveField(1) rootCollection,
  @HiveField(2) instantMix,
  @HiveField(3) recentlyPlayed,
}
```

No new fields on `MediaItemId`. Regenerate `lib/models/finamp_models.g.dart` via `build_runner` after the change.

### 2. Root menu — `music_player_background_task.dart`

Add two nodes to `_getRootMenu()`, after the existing Artists node:

```dart
MediaItem(
  id: MediaItemId(
    contentType: TabContentType.albums,
    parentType: MediaItemParentType.recentlyPlayed,
  ).toString(),
  title: _appLocalizations?.recentlyPlayedAlbums ?? "Recently Played Albums",
  playable: false,
),
MediaItem(
  id: MediaItemId(
    contentType: TabContentType.artists,
    parentType: MediaItemParentType.recentlyPlayed,
  ).toString(),
  title: _appLocalizations?.recentlyPlayedArtists ?? "Recently Played Artists",
  playable: false,
),
```

No `CONTENT_STYLE_*` extras — list view (default) is appropriate for a short recently-played list.

### 3. Data fetching — `android_auto_helper.dart`

Add a branch in `getMediaItems()`:

```dart
if (itemId.parentType == MediaItemParentType.recentlyPlayed) {
  return _getRecentlyPlayedItems(itemId);
}
```

New private method `_getRecentlyPlayedItems(MediaItemId itemId)`:

**Offline:**
```dart
return [
  MediaItem(id: 'recently_played_offline', title: 'Not available offline', playable: false),
];
```

**Online — albums** (`contentType == TabContentType.albums`):
```dart
final result = await _jellyfinApiHelper.getItemsWithTotalRecordCount(
  parentItem: _finampUserHelper.currentUser?.currentView,
  includeItemTypes: BaseItemDtoType.album.jellyfinName,
  sortBy: "DatePlayed",
  sortOrder: "Descending",
  filters: "IsPlayed",
  limit: 20,
);
```

**Online — artists** (`contentType == TabContentType.artists`):
```dart
final result = await _jellyfinApiHelper.getItemsWithTotalRecordCount(
  parentItem: _finampUserHelper.currentUser?.currentView,
  includeItemTypes: BaseItemDtoType.artist.jellyfinName,  // "MusicArtist"
  artistType: ArtistType.albumArtist,  // routes to /Artists/AlbumArtists endpoint
  sortBy: "DatePlayed",
  sortOrder: "Descending",
  filters: "IsPlayed",
  limit: 20,
);
```

Results are mapped with the existing `queueService.generateMediaItem(item, parentType: MediaItemParentType.collection, ...)` — identical to the letter-browsing code. This gives each item `playable: true` and encodes `parentType: collection` + `itemId: <jellyfin id>` in its ID.

### 4. Playback — no changes needed

Tapping a recently-played item calls the existing `playFromMediaId` path. Items with `parentType: collection` and a valid `itemId` already route through `getBaseItems(itemId)` → `queueService.startPlayback(...)`. Albums and artists both pass `_isPlayable`, so no guard changes are required.

## Out of scope

- Offline fallback via local `PlaybackHistoryService` (session-only, track-level — not worth the complexity for a first pass)
- Pagination (20 items is sufficient and avoids IPC complexity)
- Letter browsing on the recently-played list
- Recently played tracks or playlists
