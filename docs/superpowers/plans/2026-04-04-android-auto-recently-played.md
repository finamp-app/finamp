# Android Auto — Recently Played Albums Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a "Recently Played Albums" root node to the Android Auto browse tree showing up to 20 albums sorted by most recently played, with immediate playback on tap.

**Architecture:** Add a `recentlyPlayed` value to `MediaItemParentType`, regenerate the Hive adapter, wire up one root menu node, and implement `_getRecentlyPlayedItems()` in `AndroidAutoHelper`. Jellyfin only tracks `DatePlayed` on tracks (not albums), so the method fetches recent tracks, deduplicates by `albumId`, then fetches those albums by ID. Playback re-uses the existing `collection` parent type path — no changes to `playFromMediaId`.

**Tech Stack:** Flutter/Dart, Hive (local storage), `build_runner` (code generation), Jellyfin REST API via Chopper, `audio_service` package for Android Auto media browser.

## Task 2: Implement `_getRecentlyPlayedItems()` in `AndroidAutoHelper`

**Files:**
- Modify: `lib/services/android_auto_helper.dart`

- [ ] **Step 2.1 — Add the `_getRecentlyPlayedItems` method**

  In `lib/services/android_auto_helper.dart`, add the following private method. A good place is directly above or below `_fetchRootPage()` (around line 162):

  ```dart
  /// Returns up to 20 recently played albums or artists (online only).
  /// In offline mode returns a single non-playable placeholder.
  Future<List<MediaItem>> _getRecentlyPlayedItems(MediaItemId itemId) async {
    final queueService = GetIt.instance<QueueService>();

    if (FinampSettingsHelper.finampSettings.isOffline) {
      return [
        MediaItem(
          id: 'recently_played_offline',
          title: AppLocalizations.of(GlobalSnackbar.materialAppScaffoldKey.currentContext!)
                  ?.notAvailableOffline ??
              'Not available offline',
          playable: false,
        ),
      ];
    }

    final parentItem = _finampUserHelper.currentUser?.currentView;

    final QueryResult_BaseItemDto result;
    if (itemId.contentType == TabContentType.artists) {
      result = await _jellyfinApiHelper.getItemsWithTotalRecordCount(
        parentItem: parentItem,
        includeItemTypes: BaseItemDtoType.artist.jellyfinName,
        artistType: ArtistType.albumArtist,
        sortBy: 'DatePlayed',
        sortOrder: 'Descending',
        filters: 'IsPlayed',
        limit: 20,
      );
    } else {
      // albums
      result = await _jellyfinApiHelper.getItemsWithTotalRecordCount(
        parentItem: parentItem,
        includeItemTypes: BaseItemDtoType.album.jellyfinName,
        sortBy: 'DatePlayed',
        sortOrder: 'Descending',
        filters: 'IsPlayed',
        limit: 20,
      );
    }

    final List<MediaItem> mediaItems = [];
    for (final item in result.items ?? []) {
      final mediaItem = await queueService.generateMediaItem(
        item,
        parentType: MediaItemParentType.collection,
        parentId: item.parentId,
        isPlayable: _isPlayable,
      );
      mediaItems.add(mediaItem);
    }
    return mediaItems;
  }
  ```

## Notes

- **`notAvailableOffline` l10n key:** Step 2.2 asks you to check this. If missing, fall back to a hardcoded string — don't add a new ARB key just for this.
- **`QueryResult_BaseItemDto` import:** This type is returned by `getItemsWithTotalRecordCount`. It lives in `lib/services/jellyfin_api_helper.dart` and should already be imported.
- **DHU section headers:** Android Auto may auto-inject alphabetical section headers on the recently-played list if it detects alphabetical sorting. This is harmless but worth noting if it appears — it cannot be suppressed via public API (see `docs/android_auto_browsing.md`).
